## Requirements

RadioCLI needs:

- Node.js 22 or newer
- `mpv` for the best playback experience; RadioCLI expects one local playback
  backend at runtime
- `ffplay` from FFmpeg as an optional fallback

## NPM

NPM installs RadioCLI and its JavaScript dependencies. It does not install native
system playback tools like `mpv` or FFmpeg.

```bash
npm install -g radiocli
radiocli doctor
radiocli
```

Run `radiocli doctor` any time you want to verify local playback readiness.

## macOS With NPM

```bash
brew install mpv
npm install -g radiocli
radiocli
```

Install FFmpeg separately only if you want `ffplay` fallback support:

```bash
brew install ffmpeg
```

## Linux With NPM

```bash
sudo apt install mpv
npm install -g radiocli
radiocli
```

Install FFmpeg separately only if you want `ffplay` fallback support:

```bash
sudo apt install ffmpeg
```

CI covers command-mode typecheck, tests, builds, and package checks on Ubuntu.

## Homebrew Tap

The repo includes a Homebrew formula template in `packaging/homebrew` for the
native one-command macOS lane. Once the tap formula is published, the intended
install path is:

```bash
brew install ciphore/tap/radiocli
```

That formula depends on `node` and `mpv`, so native playback stays in the native
package manager instead of being installed by npm lifecycle scripts.

## Windows

Native Windows terminals are not release-tested yet. Use WSL with Linux `mpv` /
`ffplay` for the supported path.

## Local Checkout

```bash
git clone https://github.com/Ciphore/RadioCLI.git
cd RadioCLI
npm ci
npm run build
npm link
radiocli
```

If you do not want to link the package globally:

```bash
npm run dev
```

## Built Binary

After a local build, the same command-mode actions can be run directly:

```bash
node dist/cli.js check
node dist/cli.js search "lagos talk"
```

## Data Paths And Migration

The npm package and executable are both `radiocli`. Current installs use RadioCLI
storage and cache paths such as `radiocli.json` and `radiocli-cache.json`.

Existing Radio Atlas data is still discovered when a new RadioCLI store does not
exist, and legacy `RADIO_ATLAS_HOME` / `RADIO_ATLAS_DISABLE_ANIMATION` environment
variables remain supported as migration fallbacks. New automation should use
`RADIOCLI_HOME` and `RADIOCLI_DISABLE_ANIMATION`.
